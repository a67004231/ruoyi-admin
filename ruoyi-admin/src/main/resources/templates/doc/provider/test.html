概述

本章节主要描述几个采购商向万惠充平台(下面称为万惠充)下单需要使用到的接口。采购商向万惠充提交订单, 万惠充异步处理订单, 待有订单结果后以结果回调的方式向客户推送订单结果。客户亦可主动发起结果查询主动询问万惠充有关订单情况。

向万惠充表明要购买的商品

采购商下单时，需要向万惠充表明要下单的有关商品信息，即买什么。简单来说即向万惠充表明要下单的是哪个 sku。这个通过 goodsId、skuId、outSkuId 和 Attribute Fillings 系列参数完成指定。向万惠充表明要下单的 sku 的方法可以总结为下列几种:

  • 通过 skuId 直接指定 sku.
  • 通过 goodsId 和 outSkuId 指定 sku.
  • 通过 goodsId 和足够的 Attribute Fillings 系列参数隐式指定 sku.

如果提供的信息有多余的部分，比如 skuId, goodsId 和 outSkuId 都传入了则万惠充和核对所有信息是否一致，如果出现矛盾则下单失败。

Attribute Fillings 系列参数

Attribute Fillings 系列参数是指一系列表示下单时针对产品的属性的值填充信息。比如话费充值时，你需要向万惠充表明你要充值的面值是多少。Attribute Fillings 系列参数的参数名统一为 af.xxx。根据业务类型不同，Attribute Fillings 系列参数的定义各不相同。

话费类型的 Attribute Fillings 系列参数定义

❗️ 最简单常用的话费充值对接方式，只需传入 goodsId 指定商品，传入 af.face_value 指定面值即足以指定要下单的 sku，完成下单。

结果回调

万惠充通过结果回调的方式主动向采购方推送订单结果，当订单首次到达已发货(4)或失败(-10)时万惠充向采购方推送结果，当然前提是采购方有提供结果回调地址。下面说明关于结果回调的一些要点:

  • 回调地址可以同过两种方式告知万惠充，一是通过登录万惠充网站自行配置，二是下单时通过参数指定，详见下单接口
  • 万惠充会重复尝试向采购商推送订单结果，最多尝试3次：立即推送/首次失败5分钟后推送/二次失败15分钟后推送
  • 推送成功与失败以回应的 responseCode 为准，万惠充收到 200+（200~299） 的 responseCode 则认定为成功， 其他情况为失败, 推送成功后万惠充不再重复推送
  • 为了您的财产安全，请务必校验签名，详见签名规则
